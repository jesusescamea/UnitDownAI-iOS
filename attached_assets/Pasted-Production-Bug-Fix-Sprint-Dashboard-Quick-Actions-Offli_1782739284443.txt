ISSUE 3 — Offline Mode must actually work
==================================================

Offline is a core HVAC field requirement.

Technicians often work on rooftops, mechanical rooms, basements, hospitals, warehouses, and remote sites with poor reception.

UnitDown must allow the tech to keep working with no reception.

Required behavior:

1. Add an Offline Mode quick-access control
Place it where techs can reach it quickly:
- bottom quick-jump button
OR
- persistent bottom bar
OR
- dashboard Quick Action

Label:
Offline Mode

2. Offline Mode should not just be a screen.
The app should automatically detect offline/online status using navigator.onLine and network request failures.

3. When offline:
User can still input:
- job notes
- voice notes/transcripts if local recording is available
- measurements
- parts used
- recommendations
- photos if local storage supports it
- completion notes
- customer summary drafts
- equipment observations
- model/serial/unit info manually

4. Store offline entries locally.
Use the existing IndexedDB/offline queue if available.
If one already exists, wire the dashboard/job flow to it.
Do not create a duplicate offline system if JobMode already has one.

5. Show sync status clearly:
- Offline — saving locally
- X items pending sync
- Syncing...
- Synced
- Sync failed — retry

6. When reception returns:
- Automatically sync queued items.
- Preserve original timestamps.
- Preserve original job association.
- Do not duplicate records.
- If sync fails, keep items queued and show retry.

7. Job Mode:
Ensure offline works inside /job and /job/:id.
The tech should be able to proceed as if reception still existed.

8. Dashboard:
Show an Offline Mode status indicator:
- Online
- Offline
- Pending sync count

9. Add manual sync button:
Sync Now

10. Never block field input because of missing reception.
If an API call fails:
- save locally
- queue for sync
- tell user it will sync later
